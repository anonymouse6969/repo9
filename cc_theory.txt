

### **Amazon EC2 (Elastic Compute Cloud)**

Q1: What is Amazon EC2, and what does it stand for?**
Answer: EC2 stands for **Elastic Compute Cloud**. It is a core AWS service that provides secure, scalable, and resizable virtual servers 
(known as instances) in the cloud. It allows users to rent computing power and run applications without having to invest in and maintain physical hardware.

Q2: What is an AMI (Amazon Machine Image)?**
Answer: An AMI is a master template that contains the software configuration required to launch an EC2 instance. 
This includes the operating system (like Linux or Windows), application servers, and pre-installed software. 
You can use standard AMIs provided by AWS, community AMIs, or create your own custom AMIs.
  
Q3: What are Security Groups in AWS?**
Answer: Security Groups act as stateful, virtual firewalls for your EC2 instances. They control incoming (inbound) and outgoing (outbound) network traffic. You configure specific rules (e.g., allowing SSH on port 22 or HTTP on port 80) to grant access. By default, all inbound traffic is blocked, and all outbound traffic is allowed.

Q4: What is the difference between stopping and terminating an EC2 instance?**
**Answer:**
* **Stopping:** The instance undergoes a normal shutdown process and enters a paused state. You stop paying for the compute hours, but the attached persistent storage (EBS volume) remains, and you can restart the instance later.
* **Terminating:** The instance is permanently deleted and cannot be restarted. By default, the root storage volume attached to it is also permanently deleted.

### **Salesforce.com & Apex Programming**

* **Q5: What is Apex, and which programming language is it most similar to?**
**Answer:** Apex is a strongly-typed, object-oriented programming language created by Salesforce. It is designed to execute centrally on Salesforce servers and allows developers to add business logic to most system events. Its syntax and structure are heavily based on **Java**.

* **Q6: What method do you use to print a message like "Welcome to Apex Programming" to the console?**
**Answer:** You use the `System.debug()` method.
*Code example:* `System.debug('Welcome to Apex Programming');`

* **Q7: Where did you execute your Apex code to perform these arithmetic operations?**
**Answer:** Ad-hoc Apex code is typically executed in the **Execute Anonymous** window, which is accessed through the **Developer Console** in the Salesforce platform.

* **Q8: What data types did you use to perform addition, subtraction, multiplication, and division?**
**Answer:** The primary numeric data types used in Apex are:
* **Integer:** For whole numbers without a decimal point (e.g., 5, 10).
* **Decimal:** For numbers requiring exact precision and fractional values (often used for currency or division results).
* **Double:** For 64-bit numbers with fractional parts, though `Decimal` is more commonly used in Salesforce for exact calculations.


* **Q9: When writing an Apex program to display a multiplication table, which control flow statements did you use?**
**Answer:** A loop structure is required. A standard `for` loop is most common.
*Code example:*
`for(Integer i = 1; i <= 10; i++) {`
`    System.debug(5 * i);`
`}`

* **Q10: What are Governor Limits in Salesforce Apex?**
**Answer:** Governor Limits are strict execution limits enforced by the Salesforce platform. Because Salesforce is a multi-tenant environment, these limits prevent a single user's code from monopolizing shared server resources (like memory, database connections, or CPU time). If code exceeds a limit, the transaction is immediately halted and rolled back.
* **Q11: What does "multi-tenant architecture" mean in the context of Salesforce?**
**Answer:** Multi-tenant architecture means that a single instance of the software and its underlying infrastructure serves multiple customers (tenants). While everyone shares the same server and database resources, each tenant's data is strictly isolated and secure, completely invisible to other organizations.

### **Salesforce Cloud (Lightning Platform)**

* **Q12: What is the Salesforce Lightning Platform?**
**Answer:** The Lightning Platform is a Platform as a Service (PaaS) that enables developers to build, test, and deploy custom applications on Salesforce quickly. It utilizes a modern, component-based user interface framework (Lightning Web Components) that works seamlessly across desktop and mobile devices.

* **Q13: What is the difference between a Standard Application and a Custom Application in Salesforce?**
**Answer:**
* **Standard Applications:** These are pre-packaged apps provided by Salesforce out-of-the-box, such as the Sales app or Service app.
* **Custom Applications:** These are apps built by administrators or developers within an organization to meet specific, unique business needs that aren't covered by standard apps.


* **Q14: Explain the MVC architecture as it applies to Salesforce.**
**Answer:** Salesforce applications are built on the Model-View-Controller (MVC) architecture:
* **Model (Data):** Represents the database. In Salesforce, this consists of Standard and Custom Objects, and Fields.
* **View (UI):** Represents the user interface. In Salesforce, this is handled by Lightning Pages, Lightning Components, or Visualforce.
* **Controller (Logic):** Handles the business logic. In Salesforce, this is managed by Apex Classes and standard controllers.


* **Q15: What are Custom Objects and Custom Fields?**
**Answer:** In database terms, Custom Objects are custom database tables that you create to store data specific to your company (e.g., an "Employee" or "Inventory" object). Custom Fields are the distinct columns within those tables that hold specific pieces of data (e.g., "Employee ID" or "Stock Count").

### **Amazon S3 (Simple Storage Service)**

* **Q16: What is Amazon S3, and what kind of storage does it provide?**
**Answer:** Amazon S3 (Simple Storage Service) is a highly scalable cloud storage service. 
  It provides **Object Storage**, meaning it stores data as discrete units (objects) along with their metadata and a unique identifier, rather than in file hierarchies or disk blocks. It is ideal for storing backups, media files, and static website assets.

* **Q17: What is an S3 Bucket, and what is the most important rule about naming one?**
**Answer:** An S3 Bucket is a logical container used to store objects (files). The most critical rule is that **a bucket name must be globally unique**.
  Once a bucket name is taken by any user across any AWS account worldwide, no one else can use that exact name.

* **Q18: What is the difference between Object Storage (S3) and Block Storage (EBS attached to EC2)?**
**Answer:**
* **S3 (Object Storage):** Stores files in a flat namespace over the web, accessed via API calls (HTTP/HTTPS). You cannot install an operating system on it.
* **EBS (Block Storage):** Acts as a raw, unformatted physical hard drive that is mounted directly to an EC2 instance. It is used to boot operating systems and run databases that require low-latency file system access.

### **Amazon RDS (Relational Database Service)**

* **Q19: What is Amazon RDS, and what database engines does it support?**
**Answer:** Amazon RDS is a managed web service that simplifies the setup, operation, and scaling of a relational database in the cloud. 
  It currently supports several database engines: Amazon Aurora, PostgreSQL, MySQL, MariaDB, Oracle Database, and Microsoft SQL Server.

  * **Q20: Why would you use Amazon RDS instead of installing a database directly onto an EC2 instance?**

  **Answer:** Amazon RDS is a **fully managed service**. This means AWS automates administrative tasks such as hardware provisioning, database setup, automated backups, software patching, and failover management. If you install a database directly on an EC2 instance, you are entirely responsible for all database administration, maintenance, and operating system updates.

### **Uninformed Search (BFS & DFS)**

* **Q1: What are the primary data structures used to implement BFS and DFS iteratively?**
**Answer:** Breadth-First Search (BFS) is implemented using a **Queue** (which follows a First-In-First-Out, or FIFO, principle). Depth-First Search (DFS) is implemented iteratively using a **Stack** (which follows a Last-In-First-Out, or LIFO, principle).
* **Q2: The problem asks for a recursive DFS. How does recursion replace the need for an explicit data structure?**
**Answer:** When you use recursion for DFS, you do not need to create an explicit stack data structure in your code. Instead, the programming language's internal **system call stack** automatically keeps track of the active subroutines (the vertices currently being explored). As the recursive function calls itself, new vertices are pushed onto the system stack, and as it returns, they are popped off.
* **Q3: What is the time complexity for BFS and DFS if the graph is represented using an adjacency list?**
**Answer:** The time complexity for both algorithms is **$O(V + E)$**, where $V$ represents the total number of vertices (nodes) and $E$ represents the total number of edges. This is because every vertex is visited once, and every edge is examined once.
* **Q4: Between BFS and DFS, which one guarantees finding the shortest path in an unweighted graph?**
**Answer:** **BFS** guarantees finding the shortest path in an unweighted graph. Because it explores the graph level by level, radiating outward from the starting node, the first time it discovers the target node, it has taken the minimum possible number of steps (or edges) to reach it.

### **Informed Search (A* Algorithm)**

* **Q5: What is the evaluation function used in the A* search algorithm?**
**Answer:** The A* algorithm uses the evaluation function **$f(n) = g(n) + h(n)$**.
* $g(n)$ is the exact, known cost of the path from the start node to the current node $n$.
* $h(n)$ is the heuristic, which is an estimated cost from the current node $n$ to the goal node.
* $f(n)$ is the total estimated cost of the cheapest solution path that goes through node $n$.


* **Q6: What makes a heuristic "admissible"?**
**Answer:** An admissible heuristic is one that **never overestimates** the actual cost to reach the goal. It is always optimistic. For A* to guarantee finding the optimal shortest path, its heuristic must be admissible.
* **Q7: If you implemented A* for the 8-Puzzle, what heuristic functions could you use?**
**Answer:** The two most common heuristics are:
1. **Number of Misplaced Tiles:** Counting how many tiles are currently not in their target goal positions.
2. **Manhattan Distance:** Calculating the sum of the absolute horizontal and vertical distances of each tile from its current position to its correct goal position.


* **Q8: How does A* behave if the heuristic function $h(n)$ is always 0?**
**Answer:** If $h(n)$ is always $0$, the evaluation function simply becomes $f(n) = g(n)$. In this scenario, A* completely loses its "informed" directional guidance and degenerates into **Dijkstra's Algorithm**, exploring in all directions based purely on path cost.

### **Greedy Algorithms**

* **Q9: Define the core philosophy of a Greedy Algorithm.**
**Answer:** The core philosophy of a greedy algorithm is to make the **locally optimal choice** at each individual step or stage of a problem. It hopes that by continuously taking the immediate best option available right now, it will ultimately arrive at a globally optimal solution.
* **Q10: What is the main difference between Prim's and Kruskal's algorithms for finding a Minimum Spanning Tree (MST)?**
**Answer:**
* **Prim's Algorithm:** Builds the MST by starting with a single vertex and continually growing a single, connected tree by adding the cheapest edge that connects a visited vertex to an unvisited vertex.
* **Kruskal's Algorithm:** Sorts all edges in the entire graph by weight from smallest to largest, and then picks the smallest edges one by one to add to the MST, regardless of where they are, as long as they do not form a cycle.


* **Q11: Why is Selection Sort categorized as a greedy algorithm?**
**Answer:** Selection Sort operates greedily because, during each pass through the array, it actively searches for and selects the absolute minimum (or maximum) element from the unsorted portion and immediately places it into its final, correct position. It makes the locally optimal choice at every step.
* **Q12: Dijkstra's algorithm is mentioned alongside MST algorithms. Does Dijkstra's find the Minimum Spanning Tree?**
**Answer:** **No, it does not.** Dijkstra's algorithm is used to find the **Single-Source Shortest Path** (the shortest distance from one starting node to all other nodes in the graph). While it behaves very similarly to Prim's algorithm in how it greedily selects the next node, it is solving a different problem than an MST.

### **Constraint Satisfaction Problems (CSP) & Backtracking**

* **Q13: What are the three main components of any Constraint Satisfaction Problem (CSP)?**
**Answer:**
1. **Variables:** The entities that need to be assigned values (e.g., the regions on a map, or the queens on a board).
2. **Domains:** The specific set of possible values that can be assigned to each variable (e.g., Red, Green, Blue).
3. **Constraints:** The rules that specify which combinations of values are valid or forbidden (e.g., adjacent regions cannot be the same color).


* **Q14: How does Backtracking optimize a naive brute-force search?**
**Answer:** Backtracking optimizes search by **pruning** invalid paths early. Instead of generating every possible complete solution and then checking if it is valid (which brute-force does), backtracking builds a solution step-by-step. The moment a step violates a constraint, it abandons (backtracks from) that entire branch of possibilities, saving significant computational time.
* **Q15: What are the specific constraints for the N-Queens problem on an $N \times N$ chessboard?**
**Answer:** The constraints to ensure no two queens attack each other are:
1. No two queens can share the same **row**.
2. No two queens can share the same **column**.
3. No two queens can share the same **diagonal** (both major and minor diagonals).


* **Q16: In the context of the Graph Coloring problem, what is meant by the "chromatic number"?**
**Answer:** The chromatic number is the **minimum number of distinct colors** required to color all the vertices of a given graph such that no two adjacent vertices (nodes connected by an edge) share the same color.

### **Rule-Based Chatbots**

* **Q17: What is the fundamental difference between a rule-based chatbot and a generative AI chatbot?**
**Answer:**
* **Rule-based Chatbots:** Operate on predefined logic, decision trees, or keyword matching written by a programmer. They can only output responses that were explicitly coded into their system.
* **Generative AI Chatbots:** Use Large Language Models (LLMs) and machine learning to understand complex natural language and dynamically generate novel, human-like responses on the fly.


* **Q18: When implementing this in Python 3, how did you handle user input that doesn't exactly match your predefined rules?**
**Answer:** To prevent the program from crashing or silently failing, you implement a **"fallback" or default response**. If the input does not match any recognized keywords or regular expressions, the bot catches this and returns a standard phrase like, "I'm sorry, I didn't understand that. Could you please rephrase your question?"
* **Q19: How can you make a simple keyword-based chatbot slightly more robust against user variations (like capitalization or typos)?**
**Answer:**
1. **Case conversion:** Convert all user input to `.lower()` before checking for keywords.
2. **String manipulation:** Strip leading/trailing whitespace or remove punctuation.
3. **Regular Expressions (Regex):** Use regex to look for root words or patterns instead of exact string matches.
4. **Similarity matching:** Use libraries (like difflib or Levenshtein distance) to catch slight typos.


* **Q20: What are the main limitations of the elementary chatbot you developed?**
**Answer:**
1. **No Context/Memory:** It treats every user message in isolation and cannot remember what was said two lines ago.
2. **Brittle Logic:** It relies strictly on the user phrasing things the way the programmer expected. It struggles with synonyms or complex sentence structures.
3. **Lack of Intent Understanding:** It matches keywords, but it doesn't actually "understand" the meaning or intent behind the user's question.
